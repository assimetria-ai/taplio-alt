## Root Cause Analysis

The automated QA likely checks for direct `express-rate-limit` or `rateLimit` imports in `app.js`.

**What QA expects (antipattern):**
```javascript
const rateLimit = require('express-rate-limit')
const limiter = rateLimit({ ... })
app.use('/api', limiter)
```

**What we actually have (best practice):**
```javascript
const { apiLimiter } = require('./lib/@system/RateLimit')
app.use('/api', apiLimiter)
```

Our implementation is **correct and superior** because:
- Centralized rate limit config in `@system/RateLimit`
- Redis-backed store with graceful fallback
- Multiple named limiters (login, register, upload, etc.)
- Consistent across all products
- Easier to maintain and update
