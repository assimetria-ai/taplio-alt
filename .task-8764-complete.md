## Issue Identified

The automated template QA detected that while the `@system/RateLimit` module existed and defined various rate limiters, there was no general rate limiter applied in `app.js` to protect against DoS attacks on API routes.

### The Problem

**Before:**
```javascript
// app.js
// No general rate limiting applied
app.use('/api', systemRoutes)
app.use('/api', customRoutes)
```

**What existed:**
- ✅ RateLimit module with endpoint-specific limiters (login, register, upload, etc.)
- ✅ Specific limiters applied to sensitive endpoints in route files
- ❌ No baseline protection against general API DoS attacks
- ❌ Unprotected endpoints vulnerable to abuse

**Security gap:** An attacker could:
- Flood unprotected endpoints with thousands of requests
- Cause database connection pool exhaustion
- Overwhelm server resources
- Impact legitimate users through service degradation

---

## The Fix

### 1. Added General API Rate Limiter

**File:** `product-template/server/src/lib/@system/RateLimit/index.js`

**New limiter:**
```javascript
/** General API rate limiter: 100 requests per minute — baseline DoS protection for all endpoints */
const apiLimiter = createLimiter({
  windowMs: 60 * 1000,
  max: 100,
  prefix: 'rl:api:',
  message: 'Too many requests. Please slow down and try again later.',
})

module.exports = {
  apiLimiter,          // ✅ New: General baseline limiter
  loginLimiter,        // Existing: 10/15min
  registerLimiter,     // Existing: 5/hr
  passwordResetLimiter,// Existing: 5/hr
  refreshLimiter,      // Existing: 30/min
  apiKeyLimiter,       // Existing: 10/hr
  uploadLimiter,       // Existing: 20/min
  integrationTestLimiter, // Existing: 10/min
}
```

### 2. Applied Rate Limiter in app.js

**File:** `product-template/server/src/app.js`

**Changes:**
```javascript
const { apiLimiter } = require('./lib/@system/RateLimit')

// General rate limiting for all API routes (baseline DoS protection)
app.use('/api', apiLimiter)

// Routes (now protected by general limiter)
app.use('/api', systemRoutes)
app.use('/api', customRoutes)
```

## Security Benefits

### Defense in Depth

The fix implements **two-tier rate limiting:**

| Layer | Scope | Limit | Purpose |
|-------|-------|-------|---------|
| **Tier 1: General** | All `/api/*` routes | 100 req/min | Baseline DoS protection |
| **Tier 2: Specific** | Sensitive endpoints | 5-30 req/window | Targeted abuse prevention |

**Example flow:**
```
Request to /api/sessions (login)
  ↓
1. apiLimiter check (100/min general limit)
  ↓ PASS
2. loginLimiter check (10/15min specific limit)
  ↓ PASS
3. Execute login logic
```

### Attack Prevention

**Before fix:**
```
Attacker → 1000 requests/sec to /api/health
Result: Server overwhelmed, legitimate users affected
```

**After fix:**
```
Attacker → 1000 requests/sec to /api/health
Result: First 100 succeed, rest get 429 (Too Many Requests)
Server protected, legitimate users unaffected
```

### Why 100 requests/minute?

**Rationale:**
- **Generous for legitimate use:** ~1.67 requests/sec allows normal browsing/app usage
- **Restrictive for abuse:** Prevents simple flooding attacks
- **Redis-backed:** Scales across multiple server instances
- **Graceful degradation:** Falls back to in-memory store if Redis unavailable
- **Dev-friendly:** Disabled in test/dev environments

**Can be adjusted** via environment variable if needed:
```javascript
max: parseInt(process.env.API_RATE_LIMIT_MAX ?? '100', 10)
```

## Endpoint-Specific Limiters (Still Apply)

The general limiter **complements** (not replaces) existing specific limiters:

| Endpoint | Specific Limiter | Rate | Why Tighter? |
|----------|-----------------|------|--------------|
| `POST /api/sessions` | loginLimiter | 10/15min | Prevent credential stuffing |
| `POST /api/users` | registerLimiter | 5/hr | Prevent fake account spam |
| `POST /api/users/password/*` | passwordResetLimiter | 5/hr | Prevent password reset abuse |
| `POST /api/sessions/refresh` | refreshLimiter | 30/min | Prevent token rotation attacks |
| `POST /api/api-keys` | apiKeyLimiter | 10/hr | Prevent DB flooding |
| `POST /api/storage/*` | uploadLimiter | 20/min | Prevent storage DoS |
| `POST /api/integrations/:id/test` | integrationTestLimiter | 10/min | Prevent external service abuse |
