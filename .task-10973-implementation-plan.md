# Task #10973 - Taplio Alt MVP Implementation Plan

## Features to Implement (6 total)

### 1. Authentication ✅ (Already exists in @system)
- Email/password login+register+forgot
- Using existing @system/auth routes

### 2. Post Scheduler
**Database Tables:**
- `posts` (id, user_id, content, scheduled_for, published_at, status, created_at)
- `post_metrics` (post_id, likes, comments, shares, impressions, engagement_rate)

**API Routes (@custom/posts):**
- POST /api/posts - Create scheduled post
- GET /api/posts - List all posts (with filters: status, date range)
- GET /api/posts/:id - Get single post
- PATCH /api/posts/:id - Update post
- DELETE /api/posts/:id - Delete post
- POST /api/posts/:id/publish - Publish immediately

**Frontend Pages (@custom/pages):**
- PostScheduler.jsx - Create/edit posts with calendar picker
- PostsList.jsx - List view of all posts

### 3. Content Templates
**Database Tables:**
- `content_templates` (id, user_id, name, content, category, created_at)

**API Routes (@custom/templates):**
- POST /api/templates - Create template
- GET /api/templates - List templates
- GET /api/templates/:id - Get template
- PATCH /api/templates/:id - Update template
- DELETE /api/templates/:id - Delete template

**Frontend Pages:**
- ContentTemplates.jsx - Browse and manage templates

### 4. Engagement Analytics
**Database Tables:**
- Uses `post_metrics` table from Post Scheduler

**API Routes (@custom/analytics/engagement):**
- GET /api/analytics/engagement - Get engagement metrics (date range filter)
- GET /api/analytics/engagement/trends - Get trends over time

**Frontend Pages:**
- EngagementAnalytics.jsx - Charts and metrics display

### 5. Analytics (General)
**API Routes (@custom/analytics):**
- GET /api/analytics/overview - Dashboard overview stats
- GET /api/analytics/performance - Performance metrics

**Frontend Pages:**
- AnalyticsDashboard.jsx - Main analytics view

### 6. Content Calendar
**Frontend Pages:**
- ContentCalendar.jsx - Calendar view of scheduled posts (reuses posts API)

## Implementation Order
1. Database migrations (posts, post_metrics, content_templates)
2. API routes (posts, templates, analytics)
3. Frontend pages (one by one)
4. Testing each feature
5. Git commit + push
