## 1. Email System

### Current Implementation

**Location**: `server/src/lib/@system/Email/`

#### Features ✅

- **Multi-Provider Support**:
  - Resend (native API)
  - SMTP (any provider including Resend SMTP relay, SendGrid, etc.)
  - AWS SES
  - Console (dev fallback)
  - Automatic provider detection and failover

- **Pre-built Email Templates**:
  - Verification email
  - Password reset email
  - Welcome email
  - Invitation email
  - Magic link email
  - Notification email (generic)
  - Template preview system

- **Email Tracking & Analytics**:
  - Sent/delivered/bounced/failed tracking
  - Database logging (`email_logs` table)
  - Webhook ingestion endpoint for delivery events
  - Admin APIs for email analytics

- **API Endpoints** (`/api/email/`):
  ```
  POST /api/email/test          - Send test email (admin)
  GET  /api/email/provider      - View active provider (admin)
  GET  /api/email/verify        - Verify connectivity (admin)
  GET  /api/email/quota         - SES quota info (admin, SES only)
  ```

#### Configuration

```bash
# Environment Variables
EMAIL_PROVIDER=resend|smtp|ses|console
EMAIL_FROM="App <noreply@example.com>"
EMAIL_TRACKING_SECRET=xxx

# Resend
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
RESEND_REPLY_TO=support@example.com

# SMTP (any provider)
SMTP_HOST=smtp.resend.com
SMTP_PORT=587
SMTP_USER=resend
SMTP_PASS=your_api_key
SMTP_SECURE=true
SMTP_POOL=true

# AWS SES
AWS_REGION=eu-west-1
AWS_ACCESS_KEY_ID=xxx
AWS_SECRET_ACCESS_KEY=xxx
SES_FROM_EMAIL=noreply@example.com
```

#### Code Example

```javascript
const Email = require('./lib/@system/Email')

// Send pre-built template
await Email.sendVerificationEmail({ 
  to: 'user@example.com', 
  name: 'John', 
  token: 'abc123' 
})

// Send custom email
await Email.send({
  to: 'user@example.com',
  subject: 'Custom Subject',
  html: '<p>Custom HTML</p>',
  text: 'Custom text'
})
```

## 2. File Upload & Storage

### Current Implementation

**Location**: `server/src/lib/@system/StorageAdapter/`

#### Features ✅

- **Multi-Provider Storage**:
  - AWS S3
  - Cloudflare R2
  - Local filesystem
  - Switch via `STORAGE_PROVIDER` env var

- **Direct Browser-to-Cloud Uploads**:
  - Presigned PUT URLs (no server bandwidth usage)
  - Configurable expiration (default 300s)
  - Automatic file key generation (UUID + timestamp)
  - Public URL generation

- **File Management**:
  - Download presigned URLs
  - File deletion
  - Existence checks with metadata
  - Health monitoring for all adapters

- **API Endpoints** (`/api/storage/`):
  ```
  GET    /api/storage/health           - Health info (admin)
  GET    /api/storage/health/all       - All adapters (admin)
  POST   /api/storage/upload-url       - Get upload URL (authenticated)
  POST   /api/storage/download-url     - Get download URL (authenticated)
  DELETE /api/storage/object           - Delete file (authenticated)
  POST   /api/storage/local-upload     - Local upload endpoint (token-based)
  ```

#### Configuration

```bash
# Storage Provider Selection
STORAGE_PROVIDER=s3|r2|local

# AWS S3
AWS_REGION=eu-west-1
AWS_ACCESS_KEY_ID=xxx
AWS_SECRET_ACCESS_KEY=xxx
S3_BUCKET=my-bucket
S3_PUBLIC_URL=https://my-bucket.s3.amazonaws.com

# Cloudflare R2
R2_ACCOUNT_ID=xxx
R2_ACCESS_KEY_ID=xxx
R2_SECRET_ACCESS_KEY=xxx
R2_BUCKET=my-bucket
R2_PUBLIC_URL=https://pub-xxx.r2.dev

# Local Filesystem
LOCAL_STORAGE_PATH=/uploads
LOCAL_PUBLIC_URL=http://localhost:3000/uploads
```

#### Code Example

```javascript
const Storage = require('./lib/@system/StorageAdapter')

// Generate upload URL
const { url, key, publicUrl } = await Storage.createUploadUrl({
  filename: 'photo.jpg',
  contentType: 'image/jpeg',
  folder: 'avatars',
  expiresIn: 300
})

// Client uploads directly to `url` via PUT
// Then stores `key` in database for retrieval

// Later: get download URL
const { url } = await Storage.createDownloadUrl({ 
  key, 
  expiresIn: 3600 
})

// Delete file
await Storage.delete(key)
```

#### Client-Side Integration

```javascript
// Frontend upload flow
const response = await fetch('/api/storage/upload-url', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    filename: file.name,
    contentType: file.type,
    folder: 'uploads'
  })
})

const { url, key, publicUrl } = await response.json()

// Upload directly to storage (S3/R2/Local)
await fetch(url, {
  method: 'PUT',
  body: file,
  headers: { 'Content-Type': file.type }
})

// Save key in your database
await saveFileToDatabase({ key, publicUrl, filename: file.name })
```

## 3. Logging System

### Current Implementation

**Location**: `server/src/lib/@system/Logger/`

#### Features ✅

- **Structured Logging** via Pino:
  - JSON output in production
  - Pretty-printed in development
  - Configurable log levels (debug, info, warn, error, fatal)
  - Automatic service name and environment tagging

- **Usage**:
  ```javascript
  const logger = require('./lib/@system/Logger')
  
  logger.info('User logged in')
  logger.error({ err, userId: 123 }, 'Request failed')
  logger.debug({ data }, 'Processing item')
  ```

- **Configuration**:
  ```bash
  LOG_LEVEL=debug|info|warn|error
  SERVICE_NAME=api-server
  NODE_ENV=production|development
  ```

#### Current Limitations

- No centralized log aggregation (Datadog, Sentry, LogDNA)
- No log rotation or archival
- No built-in audit trail system (separate from application logs)
- Basic error tracking (no issue aggregation)

## Recommendations

### 🎯 High Priority

1. **Audit Trail System** ⭐
   - Database table for audit events
   - Automatic tracking of:
     - User actions (created/updated/deleted what)
     - IP address & user agent
     - Before/after data snapshots
   - Admin API for audit log viewing
   - **Why**: Essential for compliance (SOC 2, GDPR) and security investigations
   - **Competitor gap**: Most templates lack this

2. **Error Tracking Integration** ⭐
   - Sentry SDK integration (optional)
   - Environment variable toggle (`ERROR_TRACKING_PROVIDER=sentry|none`)
   - Automatic error capture and grouping
   - User context attachment
   - **Why**: Production debugging is impossible without error aggregation
   - **Industry standard**: Sentry, Datadog APM, Rollbar

3. **Email Template Builder/Editor**
   - React Email components (JSX-based templates)
   - Admin UI for template preview
   - Variable interpolation preview
   - **Why**: Makes template customization easier for non-developers

### 🔄 Medium Priority

4. **Advanced Email Features**
   - Attachment support
   - CC/BCC support
   - Batch sending API
   - Email scheduling (send later)
   - Template versioning

5. **Enhanced Storage Features**
   - Virus scanning integration (ClamAV)
   - Image optimization pipeline (Sharp, Thumbor)
   - CDN integration (CloudFront, Cloudflare)
   - File size limits per plan/tier

6. **Log Aggregation**
   - Datadog/LogDNA integration (optional)
   - Log retention policies
   - Search and filtering UI

### 📊 Low Priority

7. **Email Marketing Features**
   - Newsletter/campaign system
   - Subscriber management
   - Unsubscribe handling
   - Open rate tracking (pixel-based)

8. **Advanced File Management**
   - Folder/directory support
   - File sharing with expiring links
   - Zip/archive generation
   - Chunked uploads for large files

## Proposed Implementation Plan

### Phase 1: Audit Trail System (P0)

**Deliverables**:
1. Database migration for `audit_logs` table
2. Middleware for automatic request logging
3. Helper functions for manual event logging
4. Admin API endpoints for audit log viewing
5. Documentation and code examples

**Database Schema**:
```sql
CREATE TABLE audit_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  action VARCHAR(50) NOT NULL,  -- 'created', 'updated', 'deleted', 'viewed'
  resource_type VARCHAR(50) NOT NULL,  -- 'user', 'team', 'file', etc.
  resource_id VARCHAR(255),
  before_data JSONB,
  after_data JSONB,
  ip_address INET,
  user_agent TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_audit_logs_user_id ON audit_logs(user_id);
CREATE INDEX idx_audit_logs_resource ON audit_logs(resource_type, resource_id);
CREATE INDEX idx_audit_logs_created_at ON audit_logs(created_at DESC);
```

**API Example**:
```javascript
const AuditLog = require('./lib/@system/AuditLog')

// Manual logging
await AuditLog.log({
  userId: req.user.id,
  action: 'updated',
  resourceType: 'team',
  resourceId: team.id,
  beforeData: oldTeam,
  afterData: team,
  req  // Extracts IP & user agent
})

// Middleware (automatic)
app.use(auditMiddleware)  // Logs all POST/PUT/DELETE requests
```

### Phase 2: Error Tracking Integration (P0)

**Deliverables**:
1. Sentry SDK integration with env toggle
2. Automatic error capture in Express
3. User context attachment
4. Sourcemap upload config (CI/CD)
5. Documentation for self-hosting alternatives

**Configuration**:
```bash
ERROR_TRACKING_PROVIDER=sentry|none
SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
SENTRY_ENVIRONMENT=production|staging
```

**Code Example**:
```javascript
// server/src/lib/@system/ErrorTracking/index.js
const Sentry = require('@sentry/node')

if (process.env.ERROR_TRACKING_PROVIDER === 'sentry') {
  Sentry.init({
    dsn: process.env.SENTRY_DSN,
    environment: process.env.SENTRY_ENVIRONMENT,
    tracesSampleRate: 0.1,
    integrations: [
      new Sentry.Integrations.Http({ tracing: true }),
      new Sentry.Integrations.Express({ app }),
    ],
  })
}

// Express middleware
app.use(Sentry.Handlers.requestHandler())
app.use(Sentry.Handlers.errorHandler())
```
