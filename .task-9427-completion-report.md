## Verified Auth Features

### ✅ **Login System**
- Modern standalone login page with form validation
- OAuth integration (Google + GitHub buttons)
- 2FA support (TOTP redirect when required)
- Forgot password link
- Session cookie management
- Backend endpoint: `POST /api/sessions`

### ✅ **Registration System**
- Standalone registration page with comprehensive validation
- Password strength requirements (min 8 chars, uppercase, number)
- Password confirmation matching
- OAuth integration (Google + GitHub buttons)
- Auto-login after registration
- Backend endpoints: `POST /api/users`, `POST /api/sessions`

### ✅ **Password Reset Flow**
- **Forgot Password:** Email submission form (`POST /api/users/password/request`)
- **Reset Password:** Token-based password reset (`POST /api/users/password/reset`)
- Secure token validation
- User enumeration protection (always shows success)
- Redirects to login after successful reset

### ✅ **OAuth Implementation**
**Frontend:**
- `OAuthButtons.jsx` component with Google and GitHub
- Redirects to `/api/auth/google` and `/api/auth/github`

**Backend:** (`server/src/api/@system/oauth/index.js`)
- Google OAuth 2.0 flow
- GitHub OAuth flow  
- User account creation/linking
- Session cookie issuance
- Error handling with safe redirects
- Open redirect protection

### ✅ **Security Features**
- JWT-based session management (7-day TTL)
- HTTP-only cookies
- Secure and SameSite cookie attributes
- CSRF protection via SameSite
- Rate limiting on OAuth endpoints
- Safe redirect validation (prevents open redirects)
- User enumeration protection in password reset
