- `POST /api/invitations/accept/:token` - Accept invitation (public)
- `GET /api/invitations/pending` - Get user's pending invitations

Activity:
- `GET /api/teams/:teamId/activity` - Get activity log (paginated, filterable)
- `GET /api/teams/:teamId/activity/recent` - Get recent activity

### ✅ Permissions System (RBAC)

**Role Hierarchy:**
1. **Owner** (level 4) - Full control, cannot be removed
2. **Admin** (level 3) - Manage members, settings, content
3. **Member** (level 2) - Create and edit content
4. **Viewer** (level 1) - Read-only access

**Permission Matrix:**
- `team.read`, `team.update`, `team.delete`
- `members.read`, `members.invite`, `members.remove`, `members.update_role`
- `invitations.read`, `invitations.revoke`, `invitations.resend`
- `content.create`, `content.update`, `content.delete`
- `activity.read`

**Middleware:**
- `requireTeamMembership({ permission, minRole })` - Check team membership and permissions
- `requireTeamOwner()` - Require team ownership
- Role hierarchy enforcement (cannot modify higher roles)
