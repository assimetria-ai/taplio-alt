### Why This Fix Was Needed
The billing portal endpoint (`/api/stripe/create-portal-session`) requires `stripe_customer_id` from the subscription record to create portal sessions:

```javascript
const subscription = await SubscriptionRepo.findActiveByUserId(req.user.id)
if (!subscription?.stripe_customer_id) {
  return res.status(404).json({ message: 'No active subscription found' })
}

const session = await stripe.billingPortal.sessions.create({
  customer: subscription.stripe_customer_id,  // 👈 This field is critical
  return_url: `${process.env.APP_URL}/app/billing`,
})
```

Without persisting `stripe_customer_id`, the portal endpoint would fail.
