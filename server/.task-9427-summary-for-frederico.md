## API Documentation

### Login Flow
```bash
# 1. Login
POST /api/sessions
Body: { "email": "user@example.com", "password": "secure123" }
Response: { "user": { "id", "email", "name" } }
Cookies: access_token (15min), refresh_token (7d)

# 2. Refresh tokens
POST /api/sessions/refresh
Cookies: refresh_token (auto-sent)
Response: New tokens issued

# 3. Logout
DELETE /api/sessions
Response: { "message": "Logged out" }
```

### Registration Flow
```bash
# 1. Register
POST /api/users
Body: { "email": "user@example.com", "password": "secure123", "name": "John" }
Response: { "user": { "id", "email", "name" } }
Note: Verification email sent automatically

# 2. Verify email
POST /api/users/email/verify
Body: { "token": "token-from-email" }
Response: { "message": "Email verified successfully" }
```

### Password Reset Flow
```bash
# 1. Request reset
POST /api/users/password/request
Body: { "email": "user@example.com" }
Response: { "message": "If this email exists, a reset link has been sent" }

# 2. Reset password
POST /api/users/password/reset
Body: { "token": "token-from-email", "password": "newpassword123" }
Response: { "message": "Password reset successfully" }
```

### OAuth Flow
```bash
# Google
GET /api/auth/google
→ Redirects to Google consent
→ Returns to /api/auth/google/callback
→ Issues JWT cookie
→ Redirects to /app

# GitHub
GET /api/auth/github
→ Redirects to GitHub authorize
→ Returns to /api/auth/github/callback
→ Issues JWT cookie
→ Redirects to /app
```
