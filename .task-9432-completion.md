## Architecture & Conventions

### File Structure
All components follow the template's `@system` vs `@custom` convention:
```
client/src/app/components/@system/
├── Dashboard/
│   ├── WelcomeCard.jsx
│   ├── FiltersBar.jsx
│   ├── BulkActions.jsx
│   └── index.js (exports)
├── Onboarding/
│   ├── GuidedTour.jsx
│   ├── ProgressChecklist.jsx
│   └── index.js (exports)
└── UserSettings/
    ├── KeyboardShortcuts.jsx
    ├── DataExport.jsx
    ├── ConnectedAccounts.jsx
    └── index.js (exports)
```

### Component Patterns
Every component follows these patterns:

1. **JSDoc header:**
   ```jsx
   // @system — Component description
   // Brief explanation of what it does and when to use it.
   ```

2. **Controlled props:**
   ```jsx
   export function Component({
     value,           // Current state
     onChange,        // State updater
     onAction,        // Event handlers
     loading,         // Async states
     className        // Custom styling
   })
   ```

3. **Example usage in comments:**
   ```jsx
   // Example usage:
   // const tasks = [...]
   // <Component tasks={tasks} />
   ```

4. **Tailwind + design tokens:**
   - Uses `cn()` utility for class merging
   - All colors use Tailwind tokens (primary, muted, accent)
   - Consistent spacing (p-4, gap-3, space-y-6)

5. **Accessibility:**
   - Semantic HTML (`<button>`, `<label>`, `<input>`)
   - ARIA labels where needed
   - Keyboard navigation support
   - Focus visible states

## Migration Path for Existing Products

Products built from the template can pull these updates:

1. **Pull template changes:**
   ```bash
   git remote add template https://github.com/assimetria-ai/product-template.git
   git fetch template
   git merge template/main --allow-unrelated-histories
   ```

2. **Resolve conflicts:**
   - Accept `@system` changes from template
   - Keep `@custom` changes from product

3. **Test components:**
   - Run dev server
   - Check component imports
   - Verify styling matches

4. **Update product pages:**
   - Import new components where needed
   - Pass appropriate props
   - Test functionality
