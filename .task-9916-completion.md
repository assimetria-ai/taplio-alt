### 3. ❌ → ✅ Invitation Notifications Hook

**Problem:** No reusable hook for managing invitation state across components.

**Solution:**
- Created `useTeamInvitations()` custom hook
- Automatically fetches pending invitations on mount
- Provides invitation count for badges/notifications
- Includes `refresh()` and `acceptInvitation()` functions
- Handles loading and error states

**Files Created:**
- `client/src/app/hooks/@custom/useTeamInvitations.js`

**API:**
```javascript
const {
  invitations,        // Array of pending invitations
  invitationCount,    // Number (for badges)
  loading,            // Boolean
  error,              // String or null
  refresh,            // Function
  acceptInvitation,   // Async function
} = useTeamInvitations()
```

## Permission System (Already Implemented)

The template includes comprehensive role-based access control:

**Roles (Hierarchy):**
1. **Viewer** - Read-only access
2. **Member** - Create and edit content
3. **Admin** - Manage members, settings
4. **Owner** - Full control (cannot be removed)

**Permissions:**
- `team.read`, `team.update`, `team.delete`
- `members.read`, `members.invite`, `members.remove`, `members.update_role`
- `invitations.read`, `invitations.resend`, `invitations.revoke`
- `content.create`, `content.update`, `content.delete`
- `activity.read`

**Backend Middleware:**
- `requireTeamMembership(options)` - Verifies user is team member
- `requireTeamOwner()` - Verifies user is team owner
- `hasPermission(role, permission, customPermissions)` - Checks specific permission
