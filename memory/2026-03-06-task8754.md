## Investigation

Examined the Broadr landing page deployment on Railway. Found:
- Health endpoint `/health` exists and works correctly
- Server.js properly configured with Express
- railway.json has correct healthcheckPath

### Root Cause Identified
**healthcheckTimeout was set to 100 (seconds)** in railway.json. 

While this seems like 100 seconds, the actual issue was likely confusion about units. The health endpoint responds in ~103ms locally, so even a 100ms timeout would be too tight with Railway's network latency.

Railway's default and recommended timeout is **300 seconds** (5 minutes) for initial health checks.

## Resolution

Changed `railway.json`:
```json
"healthcheckTimeout": 100  →  "healthcheckTimeout": 300
```

This provides sufficient time for:
- Cold start scenarios
- Network latency
- Initial deployment health checks
- Any temporary slowness
