### The Task
Fix Railway health check failures for Broadr caused by PostgreSQL SSL connection issues.

### The Problem
Railway's health check endpoint (`/api/health`) was returning **503 Service Unavailable** because:
1. Health check performs database connectivity test
2. Railway's managed PostgreSQL uses **self-signed certificates**
3. Code was using strict SSL verification (`ssl: true`)
4. Self-signed certs failed verification → DB connection failed → health check failed

### The Solution (Already Applied)
**File changed:** `server/src/lib/@system/PostgreSQL/index.js`

**Before:**
```javascript
ssl: true  // ← Fails with self-signed certs
```

**After:**
```javascript
ssl: { rejectUnauthorized: false }  // ← Accepts self-signed certs
```
