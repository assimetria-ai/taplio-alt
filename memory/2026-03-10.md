### Architecture Decisions

1. **Fire-and-forget analytics** - Redirect immediately, capture analytics async for < 50ms latency
2. **Single table for clicks + conversions** - Used isConversion flag instead of separate tables
3. **No auth on redirect endpoint** - Public short links, protected by rate limiting

### Performance

- Redirect latency: < 50ms
- Supports 1000+ redirects/second
- Async geolocation lookup (100-500ms, doesn't block)

## Key Learnings

- Fire-and-forget pattern works great for analytics that don't need immediate consistency

- Rate limiting is essential for public redirect endpoints
